# 保养记录分页功能说明

## 🎯 功能概述

为保养记录查询接口添加了分页功能，支持分页查询所有保养记录和用户保养记录，提高大数据量查询的性能和用户体验。

## 📋 新增接口

### 1. 分页查询所有保养记录
**接口地址**: `GET /maintenance/all/page`

**请求参数**:
- `pageNum`: 页码（可选，默认为1，从1开始）
- `pageSize`: 每页大小（可选，默认为10，最大100）

**示例请求**:
```
GET /maintenance/all/page?pageNum=1&pageSize=5
```

**响应格式**:
```json
{
    "code": 200,
    "msg": "查询成功",
    "data": {
        "records": [...],      // 当前页数据
        "total": 25,           // 总记录数
        "pageNum": 1,          // 当前页码
        "pageSize": 5,         // 每页大小
        "totalPages": 5,       // 总页数
        "hasNext": true,       // 是否有下一页
        "hasPrevious": false   // 是否有上一页
    }
}
```

### 2. 分页查询用户保养记录
**接口地址**: `GET /maintenance/user/{userId}/page`

**请求参数**:
- `userId`: 用户ID（路径参数）
- `pageNum`: 页码（可选，默认为1）
- `pageSize`: 每页大小（可选，默认为10，最大100）

**示例请求**:
```
GET /maintenance/user/1/page?pageNum=1&pageSize=10
```

## 🛠️ 技术实现

### 1. 分页结果类 (PageResult)
```java
public class PageResult<T> {
    private List<T> records;     // 当前页数据
    private long total;          // 总记录数
    private int pageNum;         // 当前页码
    private int pageSize;        // 每页大小
    private int totalPages;      // 总页数
    private boolean hasNext;     // 是否有下一页
    private boolean hasPrevious; // 是否有上一页
}
```

### 2. 使用PageHelper分页插件
- 在Service层使用`PageHelper.startPage(pageNum, pageSize)`开启分页
- 使用`PageInfo`包装查询结果，获取分页信息
- 自动处理SQL分页查询

### 3. 参数验证
- 页码最小值为1，小于1时自动修正为1
- 每页大小范围1-100，超出范围时自动修正
- 提供合理的默认值

## 📊 分页信息说明

| 字段 | 类型 | 说明 |
|------|------|------|
| records | List<T> | 当前页的数据列表 |
| total | long | 总记录数 |
| pageNum | int | 当前页码（从1开始） |
| pageSize | int | 每页显示的记录数 |
| totalPages | int | 总页数（自动计算） |
| hasNext | boolean | 是否有下一页 |
| hasPrevious | boolean | 是否有上一页 |

## 🔧 配置说明

### PageHelper配置
项目已配置PageHelper分页插件，在`pom.xml`中：
```xml
<dependency>
    <groupId>com.github.pagehelper</groupId>
    <artifactId>pagehelper-spring-boot-starter</artifactId>
    <version>1.4.7</version>
</dependency>
```

### 分页参数限制
- 最小页码：1
- 最大每页大小：100
- 默认每页大小：10
- 默认页码：1

## 📈 性能优化

### 1. 数据库层面
- 使用PageHelper自动生成LIMIT和OFFSET语句
- 只查询当前页需要的数据，减少内存占用
- 支持大数据量查询

### 2. 应用层面
- 参数验证和自动修正，避免无效查询
- 统一的分页结果格式，便于前端处理
- 异常处理和日志记录

## 🎨 前端集成示例

### JavaScript分页处理
```javascript
// 获取分页数据
async function getMaintenanceRecords(pageNum = 1, pageSize = 10) {
    try {
        const response = await fetch(`/maintenance/all/page?pageNum=${pageNum}&pageSize=${pageSize}`);
        const result = await response.json();
        
        if (result.code === 200) {
            const pageData = result.data;
            console.log('当前页数据:', pageData.records);
            console.log('总记录数:', pageData.total);
            console.log('总页数:', pageData.totalPages);
            console.log('是否有下一页:', pageData.hasNext);
            console.log('是否有上一页:', pageData.hasPrevious);
        }
    } catch (error) {
        console.error('查询失败:', error);
    }
}

// 使用示例
getMaintenanceRecords(1, 5); // 第1页，每页5条
```

### 分页组件示例
```html
<!-- 分页导航 -->
<div class="pagination">
    <button onclick="goToPage(1)" :disabled="!pageData.hasPrevious">首页</button>
    <button onclick="goToPage(pageData.pageNum - 1)" :disabled="!pageData.hasPrevious">上一页</button>
    <span>第 {{ pageData.pageNum }} 页 / 共 {{ pageData.totalPages }} 页</span>
    <button onclick="goToPage(pageData.pageNum + 1)" :disabled="!pageData.hasNext">下一页</button>
    <button onclick="goToPage(pageData.totalPages)" :disabled="!pageData.hasNext">末页</button>
</div>
```

## 📝 注意事项

1. **数据一致性**: 分页查询过程中如果有新数据插入，可能导致数据重复或遗漏
2. **性能考虑**: 对于超大数据量，建议添加索引优化查询性能
3. **缓存策略**: 可以考虑对热点数据进行缓存
4. **监控告警**: 建议监控分页查询的性能和错误率
