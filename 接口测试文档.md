## 🚨 常见问题排查

### 问题1：连接被拒绝
**错误**: `Connection refused`
**解决**: 确保Spring Boot应用正在运行在9096端口

### 问题2：JSON格式错误
**错误**: `JSON parse error`
**解决**: 检查JSON格式是否正确，特别注意引号和逗号

### 问题3：字段验证失败
**错误**: `Validation failed`
**解决**: 检查必填字段是否为空，日期格式是否正确

### 问题4：数据库连接错误
**错误**: `Database connection failed`
**解决**: 检查数据库连接配置和数据库服务状态

### 问题5：user_id不能为空 ⚠️
**错误**: `Column 'user_id' cannot be null`
**原因**: 数据库外键约束，user_id字段不能为空
**解决方案**:
1. **确保数据库中有用户记录**:
```sql
-- 插入测试用户（如果不存在）
INSERT IGNORE INTO sys_user (id, phone, password, role_code, nickname, status, create_time) 
VALUES (1, '13800138000', 'e10adc3949ba59abbe56e057f20f883e', 'admin', '测试用户', 1, NOW());
```

2. **验证用户数据**:
```sql
SELECT * FROM sys_user WHERE id = 1;
```

3. **系统自动处理**: 如果无法获取用户ID，系统会自动使用默认用户ID（1L）

4. **使用JWT Token**（推荐）:
```bash
# 在请求头中添加Authorization
curl -X POST http://localhost:9096/maintenance/add \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer YOUR_JWT_TOKEN" \
  -d '{"productName": "测试产品", ...}'
```

### 问题6：文件上传失败
**错误**: `File upload failed`
**解决**: 
1. 检查uploads目录是否存在且有写入权限
2. 检查文件大小是否超过限制（10MB）
3. 检查文件格式是否支持
